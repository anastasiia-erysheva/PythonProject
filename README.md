# Решения задач из курса «Поколение Python»

Репозиторий содержит мои практические решения задач по программированию на Stepik. Код структурирован и отражает прогресс в изучении продвинутых концепций языка.

## 🛠 Технологии
* **Язык:** Python 3.12
* **Среда разработки:** PyCharm

## 📚 Изученные темы и модули

### 1. Продвинутые структуры данных (`collections`)
* **`Counter`** — автоматический подсчет элементов, поиск самых частотных значений (`most_common`).
* **`defaultdict`** — словари со значениями по умолчанию и обработка пропущенных ключей (`setdefault`).
* **`namedtuple`** — создание именованных кортежей для структурирования объектов.
* **`OrderedDict`** — упорядоченные словари и сортировка элементов.
* **`ChainMap`** — объединение нескольких словарей в один общий пул для быстрого поиска. При поиске ключа проверяет словари по очереди, позволяя легко управлять приоритетами и настройками.

### 2. Сериализация и хранение данных
* **`json`** — работа с текстовым форматом данных, чтение, запись и парсинг конфигурационных файлов (`profile.json`).
* **`pickle`** — бинарная сериализация объектов Python, сохранение и восстановление состояний.

### 3. Работа с файловыми архивами (`zipfile`)
* Чтение архивов, извлечение метаданных (`infolist`), работа со временем изменения файлов и программная запись/упаковка данных в `.zip`.

### 4. Ввод-вывод и базовые типы
* Потоковый ввод данных через системный модуль **`sys`** (`stdin`).
* Работа со встроенными коллекциями: списки (`list`), множества (`set`) и словари (`dict`).
### 5. Работа с датой и временем
* **`datetime`** — работа с датами, временем и временными интервалами.
* **`calendar`** — работа с календарем, определение високосных лет и вывод календарей на экран.
### 6. Обработка исключений (`try-except`)
* **`try-except`** — перехват и обработка ошибок во время выполнения программы во избежание её аварийного завершения.
* **`else`** — блок кода, который выполняется только в том случае, если в блоке `try` не возникло никаких исключений.
* **`finally`** — блок, который выполняется гарантированно в любом случае (например, для обязательного закрытия файлов).
* **Иерархия исключений** — древовидная структура встроенных ошибок в Python, где базовым классом является `BaseException`. Важно соблюдать порядок перехвата (от частных ошибок к общим).
* **Пользовательские исключения** — создание собственных классов ошибок путем наследования от встроенного класса `Exception` для обработки специфичной логики приложения.
* **LBYL vs EAFP** — две методики написания кода: *LBYL* (Look Before You Leap) — сначала проверь, потом делай; *EAFP* (Easier to Ask for Forgiveness than Permission) — сначала делай, а если возникнет ошибка, обработай её через `try-except` (стиль, рекомендуемый в Python).
* **Оператор `assert`** — инструмент для отладки кода, который проверяет истинность выражения и возбуждает ошибку `AssertionError`, если условие ложно.
